## 〇、术语一句话解释

- **Lean Canvas（精益画布）**：把"这门生意怎么成立"拆成 9 格的一张假设速记表——先列假设、再拿去验证，避免没想清就先做产品。
- **不公平优势（Unfair Advantage）**：别人很难抄、且砸钱也买不来的护城河，是模式能长期活下来的根（区别于"好点子/勤奋"这种可复制项）。
- **高风险假设（Riskiest Assumptions）**：一旦为假就会动摇整个生意的那几条核心猜测，必须最先拿去 MVP 验证。

## 一、Lean Canvas（9 模块）

### 1. 问题 Problem
科研人员在"出图—合规—复现"环节的三类高频痛点：
- **出图慢**：手动调 matplotlib / Origin 出期刊级图是小时级劳动，拖慢投稿节奏。
- **合规与复现乱**：期刊模板繁琐、脚本散落环境不一致，返工/撤稿风险高，损耗导师信任。
- **工具贵、预算受限**：Origin/MATLAB 授权贵；个人预算低，决策靠课题经费、流程慢。
- 现有替代：Origin（贵）、MATLAB（贵/门槛）、手动 matplotlib（慢）、雇人/求导师（关系成本）。

### 2. 客户细分 Customer Segments
- **核心人群**：海外科研人员（博士 / 博士后 / 早期科研工作者），处于论文撰写与可复现研究场景。
- **关键特征**：① 个人预算低、靠课题经费 ② 期刊级图表是刚需 ③ 部分非母语者有表达压力 ④ 发表周期时间紧 ⑤ 更信"科研人自己做的工具"。
- **早期采纳者**：LinkedIn/GEO 上活跃、正赶投稿、已用 AI 工具的年轻科研人。
- **付费决策者**：个人（小笔自掏）或课题组长 PI（批经费，客单价/粘性更高）。
- ⚠️ 此细分直接建立在假设 #1 之上。

### 3. 独特卖点 UVP
- **主陈述（沿用阶段 03 版本1 占位）**：我们帮助海外科研人员把出期刊级图表从小时级降到分钟级，通过 scifigure-pro 与 chartmind-ai 的 AI 出图，实现把时间还给科研本身。
- **一句话高概念**：「期刊级科研图，分钟级出图——由科研人自己做的 AI 工具。」
- 差异化支点：科研人内部人信用 + 现成产品（非从零讲痛点）。

### 4. 解决方案 Solution
- **scifigure-pro**：一键出期刊级图（现成 Ready，优先验证单点）。
- **chartmind-ai**：图表 AI 辅助（现成 Ready）。
- **期刊模板库**：自动按期刊格式合规（减少返工）。
- **可复现流水线模板**：版本2/3 的升级项，验证出真实付费后再叠加。
- **交付形态**：自助 + 内容驱动，非重服务——契合兼职 15–20h 与英文销售弱的现实。

### 5. 渠道 Channels
- 现状：几乎无私域（假设 #5 高置信），靠自建内容获客。
- 主结构：**GEO 内容引擎（被 ChatGPT/Perplexity 在科研方法问中推荐）+ LinkedIn 自动发布（弱连接信任沉淀）+ 官网落地页（sitemap）**，产品内免费层自带口碑/自助传播。
- 不依赖付费广告（低预算）；跑通成交后可小额试投。
- 详见第三节。

### 6. 收入来源 Revenue Streams
- 现实：个人预算低靠课题经费，付费闭环未跑（假设 #2）。
- 设计：免费增值 / 单次出图积分 / 课题组订阅 / 期刊模板包（详见第二节）。
- **首选首验路径**：先用 scifigure-pro 单点付费（版本1 提效型）验证"有人愿为真金白银出图"。

### 7. 成本结构 Cost Structure
- **固定**：Vercel 托管（免费层起）、域名、AI 工具订阅（约 $50–100/月）、支付费率（Waffo/Stripe 约 3–5% + 固定费）。
- **可变**：AI 出图按次调用成本、支付手续费、可选小额广告。
- **人力**：自身兼职 15–20h（机会成本），基本无雇人。
- **隐性**：硬编码路径/缺源码技术债的维护成本。
- 特征：低边际成本、产品驱动，契合低预算。

### 8. 关键指标 Key Metrics
- **北极星**：真实付费用户数 / 付费出图次数（直接验证假设 #2）。
- **漏斗**：GEO/LinkedIn 曝光 → 落地页访问 → 注册/试用 → 首次付费转化 → 复购/订阅留存。
- **效率**：CAC（应近零，靠内容自动化）、LTV、免费→付费转化率、单次出图成本。
- **订阅侧**：MRR、续费率（课题组订阅跑通后）。
- **内容侧**：GEO 被引用/推荐次数、LinkedIn 互动与点击。

### 9. 不公平优势 Unfair Advantage
- **科研人内部人信用**：本人即在读博士，懂场景痛点——最稀缺、别人抄不来的信任资产。
- **现成产品库**：scifigure-pro / chartmind-ai 已 Ready，可直接起步验证成交。
- **贯通出海流水线**：开发→部署→内容→分发→支付 已跑通，是可复制但属你的硬经验。
- **GEO 微垂直窗口**：大厂不屑做的科研微垂直，先占先得。
- **诚实营销准则**：fix_faq_misleading_claims 脚本佐证"不夸大"，构成信任资产。
- ⚠️ 其中"科研人内部人信用"直接依赖假设 #1。

## 二、收入来源重点设计（定价 / 收费思路）

**现实约束**：科研人员个人预算低、决策靠课题经费、付费闭环未跑（假设 #2 高置信）。因此以下全部为**假设**，需 MVP 优先验证。

**4 个收费元件（菜单）**
1. **免费增值 Freemium**：免费出基础图（带水印/限次），付费解锁高清/无标/批量/期刊模板——降低首触门槛、圈人。
2. **单次出图积分 Pay-per-image**：买积分包按需出图，决策成本最低——最适配"个人自掏/小笔"，最适合先验证付费意愿。
3. **课题组订阅 Lab/PI Subscription**：月/年订阅，PI 用课题经费付、多人共享——客单价与粘性最高，是**最优现金流产出**。
4. **期刊模板包 Journal Template Packs**：按期刊（Nature/Science/Elsevier 子刊等）卖一次性模板包，轻量、无续费压力。

**3 个收入结构候选（请选/改/混合，或提自有）**
- **结构 A · Freemium + 积分（个人优先）**：免费圈人 → 积分变现。优点：门槛最低、最快验证假设 #2；代价：客单价低、需走量。
- **结构 B · 课题组订阅为主（PI 经费）**：免费层养信任，主推课题组订阅拿 MRR。优点：LTV/现金流最优；代价：需触达并说服 PI，销售链路更长（对英文销售弱是挑战）。
- **结构 C · 模板包一次性买断（轻量）**：只卖期刊模板包，无订阅负担。优点：极简、交付轻；代价：无经常性收入、复购弱。
- **4. 我有自己的方案 / 组合**：最常见组合是 **A+B**（免费/积分养用户，课题组订阅主变现）；或 A+C 做轻量起步。

**首验建议**：先用 **scifigure-pro 单点付费（结构 A 的积分制）** 跑通第一笔真实外币收款，验证"科研人员愿意为出图付费"后再叠加 B/C。

## 三、渠道设计（低成本获客结构）

**现状**：无私域（假设 #5 高置信）、无现成海外关系、低预算、英文销售弱 → 获客必须**高杠杆、内容驱动、近零边际成本**。

**3 个获客结构候选（请选/改/混合，或提自有）**
- **结构 1 · GEO 内容引擎为主**：30 天内容日历 + FAQ + sitemap，目标是在 ChatGPT/Perplexity 回答"如何出期刊级图/某领域可视化"时被推荐。优点：踩中当下最新鲜窗口、被动流量、近零成本；代价：GEO 效果需时间积累、算法不可控。
- **结构 2 · LinkedIn 自动发布为主**：LinkedIn 模板 → 自动分发 → 弱连接信任沉淀。优点：已有可运行系统、科研人聚集；代价：新/低粉号起步慢、非私域、互动浅。
- **结构 3 · GEO + LinkedIn 双轮 + 产品内口碑（推荐）**：GEO 抓搜索意图流量，LinkedIn 做信任与再触达，产品免费层导出自带口碑传播。优点：互补、成本最低、贴合现有资产；代价：需同时维护两套内容节奏（靠已建自动化可承受）。
- **4. 我有自己的方案 / 组合**：如结构3 + 跑通后小额 Reddit/学术社群投放。

**渠道假设标注**：能否"近零成本获客"本身是高风险假设（见第五节 #3）。

## 四、BMC Lite 轻量校验（价值创造—传递—获取一致性）

| 维度 | 内容 | 一致性检查 |
|---|---|---|
| 价值主张 | 分钟级出期刊级图（版本1） | ✅ 与现成产品 scifigure-pro/chartmind-ai 咬合 |
| 客户 | 海外科研人员（博士/博士后） | ✅ 与利基候选1、VP 客户画像一致 |
| 渠道 | GEO + LinkedIn + 产品口碑 | ⚠️ 依赖假设 #3（低成本获客未验证） |
| 关系 | 自助 + 内容信任，无重服务 | ✅ 契合兼职/销售弱 |
| 收入 | 免费增值/积分/课题组订阅/模板包 | ⚠️ 依赖假设 #1（经费逻辑）与 #2（付费未跑） |
| 关键资源 | 科研人信用、现成产品、出海流水线 | ✅ 与阶段01资产一致 |
| 关键活动 | 内容自动化、产品运维、跑通支付 | ⚠️ 支付/部署有坑（假设 #4） |
| 成本 | 低边际、产品驱动 | ✅ 契合低预算 |
| 伙伴 | Waffo/Stripe（代收外币） | ✅ 降合规门槛 |

**校验结论**：价值创造与资源侧自洽；主要断裂点在**获取（#3）与获取价值（#1/#2）**两格——即 BMC 的"传递"与"获取"尚未被验证，须 MVP 优先补。

## 五、高风险假设 Top 5（一旦为假则动摇全局，附验证方法）

> 团队主理人指定必含 ①②③；④⑤ 由上游假设补充。

**#1 科研人员愿意为出图工具付费** —（依赖假设 #2 付费闭环未跑）
- 若为假的影响：收入格整体崩塌，整个轻量 SaaS 方向不成立。
- 验证方法：用 scifigure-pro 上线真实定价页（Waffo/Stripe 收款），放早鸟/限时价，观察首批 N 个**非补贴**真实付费；记录付费转化与客单价。

**#2 产品能独立运营变现（支付/部署有坑）** —（假设 #4）
- 若为假的影响：UVP"一键/分钟级"无法实现，变现链路断，且违背诚实边界（夸大）。
- 验证方法：真实部署 1 个产品、打通 Waffo/Stripe webhook、完成 1 笔真实外币收款；修硬编码绝对路径使跨机可部署；记录从访问到收款全链路是否跑通。

**#3 能靠 GEO/LinkedIn 低成本获客** —（假设 #5 无私域）
- 若为假的影响：无现成流量可借，需转付费投放，与低预算/兼职冲突，增长停摆。
- 验证方法：跑满 30 天内容日历，追踪 GEO 被推荐次数、LinkedIn 曝光/点击、落地页注册转化；核算 CAC 是否近零、能否带来首波注册。

**#4 最懂人群 = 科研人员（分水岭）** —（假设 #1，决定性）
- 若为假的影响：客户细分/UVP/解决方案/不公平优势/收入逻辑全部翻转，需重切利基（见第六节）。
- 验证方法：与"独立开发者"路线做 A/B 内容测试，对比互动与转化；或直接确认本人最深耕的人群身份。

**#5 兼职 15–20h/周能持续交付** —（假设 #3）
- 若为假的影响：产品迭代/内容节奏掉链，首单验证拖期，现金流更晚。
- 验证方法：设 2–3 周交付里程碑（先跑通 1 个产品支付+获客），看能否按时达成；遇博士节点（论文/答辩）时评估精力阈值。

## 六、对假设 #1 的依赖度 + 若为独立开发者模式如何变

**依赖度：极高（决定性）。** 整张画布的客户细分、UVP、解决方案、不公平优势（科研人信用）、渠道内容主题、收入来源（课题经费逻辑）全部建立在"科研人内部人"之上。假设 #1 为假 → 候选1/2 崩塌，模式需重切。

**若创始人最懂的其实是「独立开发者 / 出海 SaaS 创业者」**：模式核心翻转（不必重做流程，但画布要换核）
- **客户细分** → 出海独立开发者 / 泛 SaaS 创业者（原候选3，22/30，因缺内部人信用偏弱）。
- **UVP/产品** → 13+ 流水线脚本、批量部署 / 支付接入 / LinkedIn 分发模板变成主打（"流水线即工具"）。
- **不公平优势** → 从"科研内部人信用"转为"端到端出海流水线经验 + 批量部署/支付接入硬功夫"（仍稀缺，但信任叙事不同）。
- **渠道** → 内容主题从"科研方法"转"如何出海做 SaaS / 部署 / 收款 / 分发"。
- **收入** → 从"课题经费逻辑"转"开发者订阅 / 一次性模板包 / 部署-支付代搭服务"。
- **胜算判断**：整体下降——除非本人确为资深出海开发者（内部人信用转移），否则候选3 仍弱于当前候选1。
