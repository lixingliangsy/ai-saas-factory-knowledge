## 〇、术语一句话解释
- **利基 (Niche)**：一个足够小、小到巨头懒得进来、但你恰好能赢的细分市场入口。
- **三环合一**：把「新杠杆/元杠杆 ⊕ 边界变动带来的机会 ⊕ 你独有资源」三者重叠处，作为一人公司最稳的切入点。
- **六维评分**：用 痛点强度 / 杠杆密度 / 窗口红利 / 资源匹配 / 交付标准化 / 现金流潜力 六个维度给每个候选打分（各 1–5，满分 30）。

## 一、三环分析
### 环1：新杠杆 / 元杠杆
- 代码杠杆：19 个 Ready 产品 + 13+ 流水线脚本 + 支付模板 + Vercel 批量部署。
- 媒体杠杆：110 份营销文案 + GEO/SEO 内容引擎 + LinkedIn 自动发布系统。
- 元杠杆（AI）：AI 工具链 + 流水线自动化已贯通「开发→验证→AI内容→LinkedIn分发→追踪」。

### 环2：边界变动
- 技术边界：AI 把科研软件/建模变成一人+AI 即可做。
- 传播边界：**GEO（生成式引擎优化）是当下最新鲜的窗口**——海外科研人员开始用 ChatGPT/Perplexity 而非谷歌搜科研方法，大厂内容农场没占满。
- 架构边界：平台代收外币（Waffo/Stripe）降低合规门槛。
- 大公司为什么不来：单点科研工具市场太小、价值网络冲突、"自己人做给自己人"信任不可替代、GEO 微垂直不屑做。

### 环3：独有资源叠加
- 本身就是科研人员（在读博士）→ 最稀缺的"内部人信用"。
- 科研编程专长（DeepXDE/PyMoo 等）+ 全栈工程 = "领域×工程"稀有复合。
- 现存产品已含科研类成品：scifigure-pro、chartmind-ai。
- 限制：兼职 15–20h/周、低预算、英文销售口语弱、几乎无私域 → 利基必须高杠杆、低获客成本、产品驱动而非销售驱动。

### 三环重叠区
> 用「科研人群 + AI 可自动化的工具/内容 + GEO/LinkedIn 低成本海外触达 + 现成科研产品库(scifigure-pro/chartmind-ai)」切出的细分 = 最优重叠区。泛 SaaS（健身/效率）与纯开发者工具因缺"内部人信用"退出交集中心。

## 二、候选利基（六维评分，满分 30）

### 候选 1：科研人员「可发表级图表 / 可复现研究」工具 ★推荐
- 目标用户：需出期刊级图表、跑数值模拟、保证可复现性的博士/博士后/科研人员。
- 核心痛点：出图耗时长、Origin/MATLAB 贵或门槛高、可复现流程混乱，拖慢发表。
- 六维：痛点5 杠杆5 窗口4 资源5 标准化4 现金流3 → **26/30**
- 优势：三环全激活；scifigure-pro/chartmind-ai 已 Ready，可直接起步验证首笔成交。
- 风险：科研人员个人预算低（靠课题组经费）；支付/部署有坑需先修；兼职节奏交付慢。

### 候选 2：学术写作 / 发表辅助垂直 SaaS
- 目标用户：非英语母语科研人员，面临润色/格式/选刊压力。
- 六维：痛点5 杠杆4 窗口3 资源4 标准化4 现金流3 → **23/30**
- 风险：竞品多（Grammarly/AI 润色）；编辑非你长板；价格战。

### 候选 3：出海独立开发者「流水线即工具」产品
- 目标用户：想出海但卡部署/支付/分发的独立开发者。
- 六维：痛点4 杠杆5 窗口3 资源3 标准化4 现金流3 → **22/30**
- 风险：资源匹配最弱（非你的内部人圈）；竞争激烈；与科研主线脱节。

## 四、假设依赖显式标注（重点：假设 #1）
1. **最懂人群=科研人员 → 依赖度：极高（决定性）**。若修正为"独立开发者/泛 SaaS"，候选1/2 崩塌，候选3 上升但仍因缺内部人信用而偏弱；利基需重切到泛英语 SaaS 垂直，整体胜算下降。分水岭，请优先确认。
2. **付费闭环未跑通(高置信) → 高**。所有候选现金流分含未验证折扣。建议第一步：从现成科研产品(scifigure-pro)挑1个先跑通真实成交再放大。
3. **兼职15–20h/周 → 中**。利好高度标准化、高杠杆候选1。
4. **产品可运营性有坑 → 中**。优先选坑最少的现存科研产品；scifigure-pro 完成度高。
5. **几乎无私域(高置信) → 中**。强化 GEO/LinkedIn 内容路线。
6. **英文销售口语弱 → 低–中**。利好产品驱动/内容驱动获客。
