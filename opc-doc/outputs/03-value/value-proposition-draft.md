## 〇、术语一句话解释
- **价值主张**：你准备怎样帮这类用户把事办得更好、为什么该选你。
- **Jobs / Pains / Gains**：用户的"待办任务 / 痛点 / 期望收益"三件套。
- **价值主张画布**：把"用户画像"和"价值地图"对齐的一张表，看两边是否咬合。
- **Pain Relievers / Gain Creators**：产品怎么缓解痛点 / 放大收益。

## 一、目标细分客户（沿用阶段 02 候选 1）
- 人群：海外科研人员（博士 / 博士后 / 早期科研工作者）
- 场景：论文撰写与可复现研究
- 典型任务：出期刊级图表、跑数值模拟、保证可复现性、推进发表
- ⚠️ 此客户定义直接建立在假设 #1（最懂人群=科研人员）之上。

## 二、客户画像：Jobs / Pains / Gains
**Customer Jobs**
1. 产出期刊级图表（符合目标期刊格式）
2. 跑数值模拟 / 数据处理（PINN、InSAR、沉陷建模）
3. 管理"可复现"研究流程（代码+数据+环境一致）
4. 加速从实验到投稿的发表周期
5. 结果可视化以便展示
6. 证明产出效率，避免被质疑进度慢
7. 避免因格式/可复现被拒稿带来的信任损耗

**Pains**
1. 出图耗时长：手动调 matplotlib / Origin
2. 工具贵或门槛高：Origin/MATLAB 授权贵
3. 可复现流程混乱：脚本散落、环境不一致
4. 期刊格式合规繁琐
5. 缺现成模板，重复劳动
6. 预算受限，依赖课题经费
7. （部分人）非母语表达压力

**Gains**
1. 省时：出图从小时级降到分钟级
2. 合规可复现：一键打包"代码+数据+环境"
3. 发表提速：更早投稿、少返工
4. 低成本：比 Origin 订阅 / 雇人便宜
5. 期刊格式一键合规
6. 流程标准化、可复用、可交付

## 三、价值地图 Value Map
**Products & Services**：scifigure-pro（科研图表）、chartmind-ai（图表 AI）、可复现研究流水线模板
**Pain Relievers**：一键出期刊级图、自动按期刊模板合规、模板化复现、低价/免费替代 Origin
**Gain Creators**：秒级出图、可复现一键打包、省下发表周期、可复用模板库

---

## 四、3 个价值主张候选版本
**版本 1 · 提效型**：我们帮助海外科研人员把"出期刊级图"从小时级降到分钟级，通过 scifigure-pro + chartmind-ai 的 AI 出图，实现把时间还给科研本身。
- 优点：痛点直接、与现成产品咬合最紧、最适合首笔成交验证；代价：偏单点，需靠科研人信用拉开信任差。

**版本 2 · 结果型（合规发表）**：一键产出"直接能投稿"的合规图表与可复现数据包，通过期刊模板 + 复现流水线，实现发表提速、减少返工。
- 优点：价值锚定"发表"，客单价/粘性更高；代价：交付复杂度高，复现模板待搭建。

**版本 3 · 降风险型**：用可复现流水线模板锁死"数据-代码-环境"，避免因流程混乱导致返工/撤稿风险与导师信任损耗。
- 优点：打在"怕出错"强情绪上，信任壁垒高；代价：模板搭建验证成本高，受众相对窄。

**4. 我有自己的方案**：可组合，如"版本1 出图 + 版本3 复现"做成科研一站式。

**分析**：若目标"最快跑通首笔成交"（对应假设 #2），版本 1 风险最低、与现成产品最咬合，最适合作第一轮验证；版本 2/3 建议验证出真实付费后再叠加。

## 六、假设依赖显式标注（重点：#1、#2）
| 假设 | 依赖度 | 对价值主张的影响 |
|---|---|---|
| #1 最懂人群=科研人员 | 决定性 | 客户画像+主打产品全部建立其上；若否，整张画布重做 |
| #2 付费闭环未跑 | 高 | VP"实现收益"未验证；科研人员个人预算低、靠课题经费 |
| #3 兼职15-20h | 中 | VP 应主打高标准化/低接触交付，不宜重服务 |
| #4 产品可运营有坑 | 中 | "一键/秒级"承诺依赖支付/部署修通，否则 VP 夸大（违背诚实边界） |
| #5 无私域 | 中 | 交付须自助 + GEO/LinkedIn 获客 |
| #6 英文销售弱 | 低-中 | VP 靠科研人信用+产品力而非话术 |

**分水岭**：若你最懂的是「独立开发者」而非科研人员 → 客户画像翻转，价值地图换核（13+ 流水线脚本/部署/支付/分发模板），失去科研内部人信用这一最稀缺资产。
